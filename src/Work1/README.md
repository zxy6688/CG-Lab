## 核心原理

### 一、MVP 变换流程

三维顶点从物体空间到最终显示在屏幕上，通常会经历如下过程：

```text
模型空间 → 视图空间 → 裁剪空间 → 归一化设备坐标 → 屏幕空间
```

整体变换可写成：

```text
MVP = 投影矩阵 × 视图矩阵 × 模型矩阵
```

对于任意顶点 `v`，先写成齐次坐标：

```text
v_h = (x, y, z, 1)
```

然后进行矩阵变换：

```text
v' = MVP · v_h
```

再经过透视除法，得到归一化设备坐标：

```text
v_ndc = v' / w
```

最后映射到屏幕平面进行显示。

### 二、三维立方体构建

在选做部分中，程序定义了一个中心位于原点的三维正方体：

- 中心坐标：`(0, 0, 0)`
- 边长：`2`
- 顶点坐标范围：`[-1, 1]`

这个立方体共有：

- `8` 个顶点
- `12` 条边

渲染时不再只绘制二维三角形，而是遍历这 `12` 条边进行线框绘制，从而在屏幕上呈现出具有空间感的立体几何体。

### 三、透视投影

为了让立方体呈现更真实的三维视觉效果，程序采用了透视投影方式。

透视投影最直观的特点是：

- 近处看起来更大
- 远处看起来更小

因此，随着立方体旋转，观察者能够明显感受到前后远近带来的空间层次变化。

### 四、旋转插值

本次扩展的重要部分之一，是在程序中设置两个不同姿态的立方体：

- 姿态 A
- 姿态 B

然后在这两个姿态之间进行平滑过渡。

在实现中，我使用了**四元数球面线性插值（SLERP）**来完成旋转插值。  
相比直接对欧拉角做线性插值，SLERP 具有更平滑、更自然的旋转效果，也更适合用于三维姿态过渡的展示。

通过这一部分，可以更加直观地观察：

- 同一个立方体在两个姿态之间如何连续变化
- 三维旋转过程中空间方向如何变化
- 插值参数 `t` 对动画状态的影响

## 控制方式

程序运行后支持以下交互：

| 按键 | 功能 |
|---|---|
| `Space` | 暂停 / 继续自动动画 |
| `M` | 自动模式 / 手动模式切换 |
| `A` | 手动减小插值参数 `t` |
| `D` | 手动增大插值参数 `t` |
| `W` | 增加自动播放速度 |
| `S` | 减小自动播放速度 |
| `R` | 重置动画 |
| `Esc` | 退出程序 |

## 运行环境

推荐运行环境如下：

## 运行方法

在项目根目录下运行基础版本：

```bash
uv run python src/Work1/main.py
```

运行选做增强版：

```bash
uv run python src/Work1/cube_interp_demo.py
```

如果不使用 `uv`，也可以直接运行：

```bash
python src/Work1/cube_interp_demo.py
```
